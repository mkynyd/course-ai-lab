## 项目是什么

项目是 LumenLab 中"长期任务"的容器。每个项目独立维护:

- 一组上传资料与解析结果。
- 一组独立的对话(可随时切换历史对话)。
- 一个默认系统提示词与默认模型。
- 一组可被对话选中的文件作为上下文。

> 项目和普通聊天的区别:普通聊天无固定上下文,只依赖当前消息;项目聊天可以引用项目内的资料,并受项目配置约束。

## 项目类型

| 类型 | 用途 | 默认系统提示词侧重 |
|------|------|------------------|
| 实验 | 实验报告、调试记录 | 步骤化输出、引用资料原文 |
| 复习 | 复习提纲、错题总结 | 结构化整理、易于记忆 |
| 编程 | 代码 review、单元测试 | 代码优先、解释次之 |
| 通用 | 其他学习场景 | 中性回答 |

> 类型只决定默认提示词与快捷任务,用户可以在项目配置中随时覆盖系统提示词(当前需直接调用 API 或编辑数据库)。

## 创建项目

入口:侧边栏 **项目** → **新建项目**。

需要填写:

- 名称(必填)。
- 类型(实验 / 复习 / 编程 / 通用)。
- 使用场景描述(可选,用于 AI 生成系统提示词和快捷任务)。

默认模型硬编码为 `deepseek-v4-pro`,`thinkingEnabled` 默认为 `true`。

## 项目资料

支持的资料类型:

- 图片(MiniMax M3 OCR)。
- PDF(MiniMax M3 原生文档解析)。
- 文本/代码文件(本地读取)。

每份资料经历的状态:

1. `parsing` — 解析中。
2. `parsed` — 可被对话引用。
3. `failed` — 解析失败,可重试。

> 知识增强不会自动触发,需手动调用 `POST /api/files/:id/enhance`。

## 项目对话

- 项目内所有对话都属于该项目,带 `projectId` 字段。
- 对话上下文策略由查询类型决定,可能使用:
  - 用户选中文件(小文件直接全文加载)。
  - Agentic 文件范围选择。
  - 关键词检索或混合检索(关键词 + 向量)。
- 单个项目内可以创建多个对话,顶栏可切换。

## 项目配置

每个项目 API 层面可配置:

- 系统提示词(`systemPrompt`)。
- 默认模型(`defaultModel`)。
- 是否启用深度推理模式(`thinkingEnabled`)。

> 当前前端没有项目配置编辑 UI;创建时生成的系统提示词来自场景描述,后续修改需调用 `PATCH /api/projects/:id`。

## 快捷任务

快捷任务是一组预定义 Prompt 模板,绑定项目类型:

- 以按钮条形式显示在项目聊天输入框上方。
- 点击后发送 `hiddenPrompt`。
- 预定义任务在 `src/lib/quick-actions.ts` 中配置。

## 项目删除

- 需二次确认,级联删除项目文件、对话、Artifact。
- 文件磁盘/对象存储副本同步清理。

> "导出项目"功能尚未实现。

## API 速查

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/projects` | 项目列表 |
| `POST` | `/api/projects` | 创建项目 |
| `GET` | `/api/projects/:id` | 项目详情(含文件、对话、快捷任务) |
| `PATCH` | `/api/projects/:id` | 更新项目配置 |
| `DELETE` | `/api/projects/:id` | 删除项目 |

详情请阅读 [API 参考](../reference/api.md)。
